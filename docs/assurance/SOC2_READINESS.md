# SOC 2 Readiness (Internal)

SOC 2 reports are produced by licensed auditors. Internally, we prepare by defining controls and proving they operate over an evidence window.

## Recommended scope

Start with **Security** + **Availability**. Add Confidentiality/Privacy when contracts require.

## Readiness checklist

### Control design
- [ ] Define control objectives (what we are trying to prevent)
- [ ] Write control activities (who/what/how often)
- [ ] Assign control owners

### Control operation evidence
- [ ] Access reviews (monthly/quarterly)
- [ ] Incident response tests/tabletop
- [ ] Change management records
- [ ] Vulnerability management records and patch SLAs
- [ ] Backup/restore test records

### Evidence discipline
- [ ] Evidence is immutable and time-stamped
- [ ] Evidence is reproducible (scripts where possible)
