# Validation Framework - Quick Start Guide

## 🚀 Installation (2 minutes)

### Step 1: Copy setup script
```bash
cd /mnt/c/EdpsychConnect
bash tools/setup-validation.sh
```

This will:
- ✅ Install dependencies
- ✅ Set up git hooks
- ✅ Create directories
- ✅ Build CLI tools
- ✅ Run initial validation

### Step 2: Update package.json
```json
{
  "scripts": {
    "validate": "npx validate-code validate-dir src",
    "validate:build": "bash tools/build-validation.sh src true",
    "build": "npm run validate:build && next build"
  }
}
```

### Step 3: Test it
```bash
# Validate a file
npx validate-code validate-file src/lib/services/myService.ts

# Validate a directory
npx validate-code validate-dir src/lib/services

# Try a commit (should run pre-commit hook automatically)
git add .
git commit -m "test"
```

## 🔧 Configuration

### In Development

Enable real-time watching:

```typescript
// src/app.ts or dev-server.ts
import { createValidationWatcher } from '@/lib/validation/validationWatcher';

if (process.env.NODE_ENV === 'development') {
  createValidationWatcher({
    directories: ['src/lib/services', 'src/components'],
    debounceMs: 1500,
    onError: (error, file) => {
      console.error(`❌ ${file}: ${error.message}`);
    },
    onSuccess: (file) => {
      console.log(`✅ ${file} validated`);
    }
  }).catch(console.error);
}
```

### In package.json

```json
{
  "scripts": {
    "dev": "node -r dotenv/config src/app.ts && next dev",
    "validate": "npx validate-code validate-dir src",
    "validate:strict": "npx validate-code validate-dir src --strict",
    "validate:build": "bash tools/build-validation.sh src true",
    "build": "npm run validate:build && next build",
    "precommit": "bash tools/pre-commit-validate.sh"
  }
}
```

### In CI/CD (.github/workflows/validate.yml)

```yaml
name: Code Validation

on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run validate:build
```

## ✅ Validation Examples

### Example 1: Detect Missing Method

File: `src/lib/services/analyticsService.ts`

```typescript
export class AnalyticsService {
  async trackEvent(event: string): Promise<void> {
    // ❌ This method doesn't exist
    const result = await this._sendToServer(event);
  }
}
```

**Run validation:**
```bash
npx validate-code validate-file src/lib/services/analyticsService.ts
```

**Output:**
```
❌ ERRORS (1):
  1. UNDEFINED_METHOD [critical]
     File: src/lib/services/analyticsService.ts:5:30
     Message: Method '_sendToServer' is called but not defined
     Suggestion: Define method _sendToServer or check the method name
```

**Fix:** Add the method
```typescript
private async _sendToServer(event: string): Promise<any> {
  // Implementation
  return {};
}
```

### Example 2: Detect Orphaned Method

```typescript
export class UserService {
  // ⚠️ This method is never called
  async legacyGetUser(id: string): Promise<User> {
    // Old implementation
  }
}
```

**Output:**
```
⚠️  WARNINGS (1):
  1. UNUSED_METHOD
     File: src/lib/services/userService.ts:42
     Message: Method 'legacyGetUser' appears to be unused
```

**Fix:** Make it private or remove it
```typescript
private async _legacyGetUser(id: string): Promise<User> {
  // Mark as internal/deprecated
}
```

## 🔍 Understanding Validation Levels

### 🟢 Green (Passed)
- All methods are defined
- No undefined calls
- Code ready for deployment

### 🟡 Yellow (Warnings)
- Unused methods detected
- Missing documentation
- Naming conventions not followed
- **Action:** Review and fix before merge

### 🔴 Red (Errors)
- Undefined method calls
- Type mismatches
- Missing implementations
- **Action:** Fix immediately, cannot deploy

## 🆘 Troubleshooting

### Issue: Pre-commit hook not running

**Solution:**
```bash
# Check if hook exists
ls -la .git/hooks/pre-commit

# Make it executable
chmod +x .git/hooks/pre-commit

# Test it
git commit --no-verify  # This bypasses the hook (shows it works)
```

### Issue: Validation too slow

**Solution:** Exclude directories
```typescript
new ValidationService({
  excludePatterns: [
    'node_modules',
    '.next',
    'dist',
    'build',
    'coverage'
  ]
})
```

### Issue: False positives

**Solution:** Check method names match exactly
```typescript
// ❌ Wrong
const data = await this._gatherData(id);

async _gatherLearningData(id: string): Promise<any> {
  // Method name doesn't match!
}

// ✅ Correct
const data = await this._gatherLearningData(id);

async _gatherLearningData(id: string): Promise<any> {
  // Now it matches
}
```

## 🎯 Best Practices

1. **Run validation before committing**
   ```bash
   npm run validate
   git commit
   ```

2. **Use strict mode for builds**
   ```bash
   npm run validate:build  # Fails on warnings
   ```

3. **Watch for changes during development**
   - Start with: `npm run dev` (if watcher is configured)
   - Get real-time feedback

4. **Generate reports regularly**
   ```bash
   npm run validate > validation-report.txt
   ```

5. **Fix warnings, not just errors**
   - Warnings indicate future problems
   - Address them proactively

## 🚨 Production Checklist

Before deploying to production:

- [ ] Run `npm run validate:build` (passes)
- [ ] All error messages resolved
- [ ] Warnings reviewed and addressed
- [ ] Pre-commit hook enabled
- [ ] CI/CD validation passing
- [ ] No console.log() statements
- [ ] No TODO comments in services
- [ ] JSDoc comments added
- [ ] Code review completed
- [ ] Tests passing
