# Audit Hub (Internal)

This folder is the **single entry point** for anything audit-related: internal audit runs, findings, evidence, privacy artefacts, and external-auditor-style checklists.

> This is an **internal audit hub**, not a claim of self-certification.

## Where to start

### If you want to run an internal audit like an external auditor
- **Playbook:** `docs/assurance/INTERNAL_AUDIT_PLAYBOOK.md`
- **External-auditor-style checklist:** `docs/EXTERNAL-AUDITOR-COMPREHENSIVE-GUIDE.md`
- **Log the run:** `docs/assurance/AUDIT_RUN_LOG.md`
- **Log findings:** `docs/assurance/FINDINGS_REGISTER.md`

### If you want evidence (what we can actually prove)

## Current audit status (honest state)

- The repository contains multiple audit reports under `docs/` (historical + targeted audits).
- The **full comprehensive end-to-end walkthrough** (every section of the external auditor guide, executed and evidenced) is **not claimed as completed** by this hub.
- Completed internal audit slice(s) should always be recorded in `docs/assurance/AUDIT_RUN_LOG.md` and linked to evidence and findings.

## Conventions

- Findings IDs use: `INT-<AREA>-YYYY-NNN`
- Audit run IDs use: `RUN-YYYY-MM-DD-NN`
- Evidence IDs are tracked in: `docs/assurance/EVIDENCE_REGISTER.md`
